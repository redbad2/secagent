## IP 安全分析 SOP（7 步 + 验证清单）

---

### 步骤 1：IP 基础属性查询

调用 secagent__ip_threat、secagent__ip_org

**获取信息：** 地理位置、ASN 归属、风险等级、威胁标签、关联报告

**判断规则：**
- 有 c2/botnet 标签 → 高风险
- 有 scanner 标签 → 中风险
- 有 proxy/anonymizer 标签 → 低-中风险
- 标签置信度 < 30% → 忽略
- 云服务商 IP（AWS/GCP/阿里云）→ 中性，需结合威胁情报

---

### 步骤 2：关联域名反向查询

调用 qianxin_fdp__flint_rdata（rdata=目标IP）

**判断规则：**
- 关联域名有恶意标签 → 目标 IP 风险升高
- 关联域名都是正常业务 → 降低风险
- 关联域名数量极多（> 100）→ 共享主机，单个域名的恶意标签可能不准确
- 关联域名数量极少（< 5）→ 独立服务器，恶意标签可信度更高

---

### 步骤 3：威胁情报核查

调用 ctia_ip__ipinfo_ip、qianxin_fdp__ioc

**判断规则：**
- 多个独立来源都标记为恶意 → 高置信度
- 单一来源标记，其他来源无异常 → 低置信度
- 历史恶意但近期无活动 → 可能已清理，需持续监控
- DDoS 受害者标签 ≠ 攻击者

---

### 步骤 4：关联域名威胁情报

调用 secagent__domain_threat（批量查询关联域名）

**注意：** 每次最多查询 5 个域名，超过的分批查询

---

### 步骤 5：网络空间测绘

调用 hunter_mcp__hunter_search

**搜索语法：**
```
search="ip:\"目标IP\""
fields="ip,port,domain,url,web_title,protocol,status_code,component,asset_tag,vul_list"
page_size=100
```

**判断规则：**
- 开放端口异常（445/3389/22）→ 配置不当或扫描目标
- 服务组件有已知 CVE → 漏洞风险
- 非标准端口运行 Web 服务 → 需关注
- 大量端口开放 → 可能是扫描器或蜜罐

---

### 步骤 6：组件漏洞查询（当识别到明确服务组件时执行）

调用 ctia_exploit 工具查询对应组件的公开高危漏洞

**判断规则：**
- 识别到具体组件（如 WordPress、Portainer、1Panel）→ 查询该组件 CVE
- 服务未暴露版本号 → 漏洞部分仅作潜在风险参考
- CVSS 评分 ≥ 7.0 → 高危漏洞

---

### 步骤 7：验证清单

完成所有步骤后，必须覆盖以下信息点：

- [ ] IP 归属、ASN、风险等级
- [ ] 所有关联域名列表及安全状态
- [ ] 所有开放端口、服务、组件信息
- [ ] 漏洞核查结果（已检测 + 潜在风险）
- [ ] 最终安全评级和建议

---

### CDN/WAF 识别（降低风险）

- Cloudflare: AS13335, IP 段 104.16.0.0/12
- Akamai: AS20940, IP 段 23.0.0.0/12
- AWS CloudFront: AS16509, IP 段 13.32.0.0/12
- 网宿 CDN: wswebcdn.com 相关
- 如果 IP 属于 CDN，风险评分应大幅降低

---

### 常见误报模式

1. CDN IP 被标记为恶意 → 实际是 CDN 中继
2. 共享主机上其他域名恶意 → 目标 IP 本身安全
3. 低置信度标签（< 30%）→ 噪声
4. 历史标签残留 → 已清理但标签未更新
5. DDoS 受害者标签 → 不代表该 IP 是攻击者
