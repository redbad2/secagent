## 多源情报交叉关联分析

当分析深度为 deep 或需要交叉验证时启用此技能。核心原则：单一来源的结论不可靠，必须多源交叉验证。

---

### 步骤 1：汇总各工具返回的威胁标签

- 将 CTIA、FDP、Hunter 返回的威胁标签去重汇总
- 标注每个标签的来源（单源 vs 多源）
- 记录标签置信度（高/中/低）

**判断规则：**
- 多个独立来源都标记为恶意 → 高置信度，风险升高
- 单一来源标记，其他来源无异常 → 低置信度，需进一步验证
- 来源之间存在冲突 → 需要人工复查

---

### 步骤 2：标签一致性分析

**一致性判断：**
- CTIA 标记为恶意 + FDP WHOIS 显示新注册 + Hunter 发现关联恶意域名 → 高度一致，高风险
- CTIA 标记为恶意 + FDP WHOIS 显示正常注册 + Hunter 无关联恶意 → 不一致，需进一步验证
- CTIA 无标记 + FDP WHOIS 显示正常 + Hunter 无关联 → 一致，低风险

**冲突处理：**
- 如果 CTIA 标记为恶意但置信度低，且其他来源无异常 → 可能是误报
- 如果 CTIA 无标记但其他来源有异常信号 → 需要深入调查

---

### 步骤 3：关联查询链

**正向关联（域名 → IP → 基础设施）：**
1. 域名 → 解析 IP（PDNS）
2. IP → 威胁情报（CTIA）
3. IP → 归属组织（iporg）
4. 同 ASN 的其他资产（Hunter）
5. 关联域名的威胁情报

**反向关联（IP → 域名 → 基础设施）：**
1. IP → 反向 DNS（PDNS）
2. 反向域名 → 威胁情报（CTIA）
3. 域名 → 父域名追溯
4. 同 IP 的其他域名（Hunter）

**关键洞察：**
- 如果正向和反向关联都指向同一结论 → 高置信度
- 如果正向和反向关联存在冲突 → 需要进一步调查
- 关联链越长，置信度越低（中间环节可能有误）

---

### 步骤 4：时间线分析

**关键时间点：**
- 域名注册时间
- 域名首次出现在 PDNS 中的时间
- CTIA 首次标记为恶意的时间
- 最近一次活跃的时间

**时间线判断：**
- 新注册域名（< 7 天）+ 立即被标记为恶意 → 高度可疑
- 域名注册多年，最近才被标记 → 可能是域名被劫持或滥用
- 域名曾经恶意，最近无活动 → 可能已清理，但需持续监控
- 域名频繁更换 IP → 可能在规避封锁

---

### 步骤 5：基础设施特征分析

**CDN/WAF 判定：**
- 解析到 Cloudflare/Akamai/AWS WAF → 降低基础设施风险
- 但 CDN 后面的实际服务器仍需评估

**共享主机 vs 独立服务器：**
- 同一 IP 上有大量不相关域名 → 共享主机
- 同一 IP 上只有少量域名 → 独立服务器
- 共享主机上的恶意标签可能属于其他域名

**ASN 类型：**
- 数据中心 ASN → 通常是中性
- ISP ASN → 通常是中性
- 已知恶意 ASN → 风险显著升高

---

### 步骤 6：输出关联报告

**报告结构：**
1. 威胁关联图谱（文字描述各实体之间的关系）
2. 各维度风险评分汇总
3. 最终综合风险等级和置信度
4. 关键发现列表
5. IOC 列表（如有）
6. 处置建议

**置信度计算：**
- 多源一致 + 高置信度标签 → 0.8 - 1.0
- 多源一致 + 中置信度标签 → 0.6 - 0.8
- 单源标记 + 其他来源无异常 → 0.3 - 0.5
- 来源冲突 → 0.0 - 0.3

---

### 常见关联陷阱

1. **CDN 遮蔽**：域名解析到 CDN，无法看到真实服务器 IP
2. **共享主机干扰**：同 IP 上其他域名的恶意标签影响判断
3. **历史残留**：域名/IP 曾经恶意但已清理，标签未更新
4. **低置信度噪声**：CTIA 置信度 < 30% 的标签基本是噪声
5. **受害者混淆**：IP 是 DDoS 攻击的目标，不是攻击者
